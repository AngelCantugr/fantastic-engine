**Time to productive use: ~30 minutes**

## ‚ö° Fast Track Setup

### 1. Install Ollama (5 mins)

```bash
# Install Ollama
curl -fsSL https://ollama.com/install.sh | sh

# Start the server (in a separate terminal)
ollama serve

# Pull your first model
ollama pull llama3.2
```

**Verify:** Open http://localhost:11434 - you should see "Ollama is running"

---

### 2. Install Goose (2 mins)

Choose one:

**CLI (recommended for this experiment):**
```bash
npm install -g goose-ai
```

**Desktop App:**
Download from: https://github.com/block/goose/releases

---

### 3. Configure Goose (5 mins)

```bash
# Create config directory
mkdir -p ~/.config/goose

# Copy example config
cp config.example.yaml ~/.config/goose/config.yaml

# Edit if needed (defaults should work)
# nano ~/.config/goose/config.yaml
```

---

### 4. Setup MCP Server (5 mins)

```bash
# Install the Fetch MCP server
pip install mcp-server-fetch

# Verify installation
python -m mcp_server_fetch --version
```

---

### 5. Test Your Setup (5 mins)

```bash
# Start Goose CLI
goose

# Try a simple prompt:
# "What model are you using?"

# Test web fetching (if MCP configured):
# "Fetch the latest information about Ollama from ollama.com"

# Exit Goose
# Type: exit
```

## üéØ Your First Documentation Task (10 mins)

### Task: Create a documentation index

```bash
# Navigate to your learning-depot directory
cd /path/to/learning-depot

# Start Goose in that directory
goose

# Prompt:
# "Scan this directory for all markdown files and create an index.md
#  with categorized links. Organize by subdirectory."

# Review the output
cat index.md

# If good, commit it!
git add index.md
git commit -m "Add documentation index generated by Goose"
```

## üõ†Ô∏è Troubleshooting

### Ollama won't start
```bash
# Check if already running
ps aux | grep ollama

# Check port 11434
lsof -i :11434

# Try different port
OLLAMA_HOST=127.0.0.1:11435 ollama serve
```

### Goose can't connect to Ollama
```bash
# Verify Ollama is accessible
curl http://localhost:11434/api/tags

# Check Goose config
cat ~/.config/goose/config.yaml

# Ensure host is set to: http://localhost:11434
```

### MCP server not working
```bash
# Verify installation
pip list | grep mcp-server

# Reinstall if needed
pip install --upgrade mcp-server-fetch

# Check Goose config has MCP section
```

### Model is slow/not responding
```bash
# Check Ollama logs
journalctl -u ollama -f

# Try a smaller model
ollama pull llama3.2:1b

# Update Goose config to use smaller model
```

## üéì Pro Tips

1. **Start small** - Don't let Goose edit your entire repo at once
2. **Review everything** - AI makes mistakes, always check outputs
3. **Use version control** - Commit before letting Goose make changes
4. **Time-box tasks** - Set 30-minute limits
5. **Keep prompts specific** - Vague prompts = vague results
