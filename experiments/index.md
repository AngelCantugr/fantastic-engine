## ğŸ“ Experiment Best Practices

### âœ… Do

- **Document as you go** - Don't wait for "later"
- **Time-box** - Set a time limit for experiments
- **Clear goal** - Know what you're exploring
- **Small scope** - Start small, expand if promising
- **Fail fast** - If it's not working, document why and move on

### âŒ Avoid

- **Scope creep** - Stick to original goal
- **Perfection** - Done > Perfect for experiments
- **Multiple simultaneous** - Finish or pause one first
- **No documentation** - Always document learnings
